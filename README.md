# GottaGoFast

BEST::HACKath0n 2026 — Тестове завдання

## Мета роботи

Розробити програмне рішення для автоматизованого розбору лог-файлів польотних контролерів Ardupilot, візуалізації маршруту та обчислення підсумкових метрик польоту. Інструмент має конвертувати сирі дані з датчиків (GPS, IMU) у зручний для аналізу формат та побудувати 3D-модель переміщення апарату.

## Як запустити
1. `cd main`
2. Скачати всі необхідні бібліотеки
   `pip install -r requirements.txt`
3. `streamlit run dashboard.py`

У нас є два варіанти інтерфейсу: з аі асистентом і без(файли називаються відповідно).
Для того, щоб запустити без асистента, потрібно ввести `streamlit run dashboard_no_ai.py`.
API ключ уже зберігається в secrets.toml локально.
Для того, щоб запустити програму у себе потрібно згенерувати ключ на сайті GroqCloud,
у файлі dashboard.py знайти змінну groq_api_key і призначити їй власний АРІ ключ.

## Telemetry Parsing Core

Модуль обробки телеметрії є інтелектуальним ядром проєкту, що забезпечує перетворення "сирих" бінарних логів Ardupilot у структурований вигляд. Модуль спроєктований із фокусом на точність обчислень та чистоту вхідних даних для подальшої 3D-візуалізації.

### **Функціональні можливості**

* **Протокольна сумісність:** Повна підтримка парсингу бінарних повідомлень через `pymavlink` (типи `GPS`, `IMU`, `IMU2`, `IMU3`).
* **Інтелектуальна фільтрація (Signal Quality):** * Динамічна перевірка якості GPS-фіксу: відсіювання точок із низькою кількістю супутників (< 6) та високим показником HDOP (> 5.0).
    * Це гарантує відсутність "стрибків" на 3D-моделі траєкторії.
* **Математична підготовка:**
    * Конвертація координат із `integer` (1e7) у формат `WGS-84`.
    * Нормалізація одиниць вимірювання (висота в метри, швидкість у м/с).

### **Технологічний стек**

| Інструмент | Роль у модулі |
| :--- | :--- |
| **Pymavlink** | Ефективне потокове читання бінарних логів без втрати кадрів. |
| **Pandas** | Формування оптимізованих `DataFrame` для миттєвої передачі в ядро аналітики. |
| **NumPy** | Швидкісна векторна обробка масивів даних сенсорів. |

### **Використання**

Парсинг логу та вивід метаданих (частота Гц, тривалість, одиниці виміру)
python bin_parser.py flight.tlog --save

## Drone Flight Analyzer — Web UI + AI Assistant

Веб-застосунок для аналізу польотних логів Ardupilot з AI-звітом про аномалії.

### Web UI

- Реалізовано на Streamlit
- Завантаження `.bin` файлу через `st.file_uploader`
- Автоматичне відображення метрик польоту у вигляді карток (тривалість, дистанція, швидкість, висота)
- Розгортання детальних метрик (вертикальна швидкість, прискорення, швидкість з IMU)
- Кнопка для генерації AI-звіту

### AI-асистент

- Використано Groq API + модель LLaMA 3
- На основі обчислених метрик формується промпт українською мовою
- AI аналізує дані та повертає текстовий висновок про аномалії, оцінку польоту та рекомендації
- API ключ зберігається в `.streamlit/secrets.toml` (безпечно, не вимагає введення від користувача)

# Kinematic Analytics Core

Модуль аналітики (**analytics.py**) приймає очищені дані від парсера (GPS та IMU) і за допомогою векторних обчислень та чисельних методів перетворює "сирі" показники сенсорів на готові фізичні метрики. Розраховані дані є базою для побудови 3D-візуалізації та генерації висновків AI-асистента.

### **Функціональні можливості**

## GPS Аналітика та Глобальна Кінематика

Оскільки GPS-дані мають дискретний характер та можуть містити шуми вимірювання, модуль використовує векторизовані геодезичні обчислення для отримання точних метрик руху.

### **1. Загальна дистанція та формула Haversine (total_distance)**

Для обчислення відстані між точками на сферичній поверхні Землі (WGS-84) використовується формула гаверсинусів. Це дозволяє уникнути великих похибок, які виникають при використанні звичайної Евклідової метрики на великих відстанях.

* **Векторизація:** Обчислення проводяться за допомогою NumPy над усім масивом координат одночасно, що забезпечує миттєву обробку навіть довгих логів.
* **Математична модель:**
  $$d = 2r \arcsin\left(\sqrt{\sin^2\left(\frac{\Delta\phi}{2}\right) + \cos(\phi_1)\cos(\phi_2)\sin^2\left(\frac{\Delta\lambda}{2}\right)}\right)$$
  *Де $\phi$ — широта, $\lambda$ — довгота, $r$ — радіус Землі (6,371,000 м).*

### **2. Горизонтальна швидкість (max_horizontal_speed)**

Метрика обчислюється як перша похідна відстані по часу між сусідніми валідними GPS-точками.

* **Конвертація одиниць:** Результат переводиться з системних м/с у зручні для аналізу км/год.
* **Фільтрація:** Алгоритм автоматично ігнорує записи з нульовим або від'ємним кроком часу ($\Delta t \le 0$) для запобігання помилкам ділення на нуль.

### **3. Вертикальна швидкість та Rolling Median (max_vertical_speed)**

Обчислення швидкості підйому/спуску є найбільш критичним до шумів барометричного висотоміра. Для отримання адекватних цифр застосовано двостадійну обробку:

* **Згладжування (Median Filter):** Використовується ковзне медіанне вікно розміром 11 семплів. Це ефективно відсікає імпульсні шуми висотоміра, не "замилюючи" при цьому реальні піки швидкості.
* **Фізичне обмеження:** Програмно відсікаються вертикальні швидкості понад 50 м/с, які є фізично неможливими для цивільних БПЛА і свідчать про помилки сенсора.

### **4. Тривалість польоту (flight_duration)**

Визначає чистий час активності системи, обчислюючи різницю між часовими мітками (timestamps) останнього та першого отриманих пакетів даних.

* **Точність:** Час розраховується з точністю до мікросекунд (на рівні логу) і округлюється до двох знаків після коми для фінального звіту.

## IMU Аналітика

Оскільки акселерометри схильні до накопичення шумів, для розрахунку показників застосовано спеціальний математичний апарат.

### **1. Максимальне прискорення (get_max_acceleration)**

Датчик фіксує прискорення по трьох осях, тому для знаходження реальної сили застосовується обчислення довжини 3D-вектора.

* **Компенсація гравітації:** Оскільки акселерометр завжди фіксує тяжіння Землі (~9.81 м/с²), функція очищає результат від цього впливу.
* **Математична модель:**
  $$|a_{net}| = \left| \sqrt{a_x^2 + a_y^2 + a_z^2} - 9.81 \right|$$

### **2. Максимальна швидкість через Leaky Integrator (get_max_speed_imu)**

Класичне інтегрування ($v_i = v_{i-1} + a_i \cdot \Delta t$) призводить до необмеженого накопичення похибки (дрейфу), де швидкість зростає лінійно відносно часу. Для стабілізації результатів застосовано **Leaky Integrator** (інтегратор із витоком).

* **Усунення статичного зміщення (Bias Removal):** Перед початком обчислень алгоритм калібрується за першими 50 відліками (~0.5 сек). Припускаючи, що апарат у цей момент нерухомий, обчислюється середній вектор шуму $b = \text{mean}(a_{0:50})$, який згодом віднімається від усіх вимірів.
* **Метод "забування" (Forgetting Factor):** Впроваджено коефіцієнт згасання $\gamma = 0.995$. Це обмежує накопичення похибки певним сталим значенням (steady-state bias response).
* **Часова константа:** При частоті опитування $\sim 100$ Гц ($\Delta t \approx 0.01$ с), часова константа фільтра становить $\tau \approx \frac{\Delta t}{1-\gamma} \approx 2$ секунди, що дозволяє відстежувати реальну динаміку польоту, ігноруючи довготривалий дрейф.
* **Формула кроку:**
  $$v_i = \gamma \cdot v_{i-1} + a_{corrected} \cdot \Delta t_i$$
